# 快速开始 - 5 分钟上手指南

## 🎯 目标

5 分钟内完成：上传场景代码 → 验证 → 预览的完整流程

## 📋 前置要求

- ✅ Python 3.7+
- ✅ Node.js 14+
- ✅ 已安装项目依赖 (`npm install`)

## 🚀 步骤 1: 启动服务器（1 分钟）

### macOS / Linux

```bash
cd web-uploader
chmod +x start.sh
./start.sh
```

### Windows

```bash
cd web-uploader
start.bat
```

### 手动启动

```bash
cd web-uploader
pip install flask flask-cors
python server.py
```

**看到这个输出表示成功**：
```
============================================================
🚀 场景上传管理器 API 服务器
============================================================
🔗 访问地址: http://localhost:8000
🌐 前端界面: http://localhost:8000/
============================================================
```

## 🐛 常见问题

### Q1: 服务器启动失败？

**检查端口占用**：
```bash
# macOS/Linux
lsof -i :8000

# 如果被占用，杀死进程
kill -9 <PID>
```

### Q2: 预览启动失败？

**检查 Node.js 和依赖**：
```bash
node --version  # 应该 >= 14
npm --version

# 重新安装依赖
cd /path/to/project
npm install
```

### Q3: 代码验证失败？

**常见错误**：
- ❌ 缺少 `import React from "react"`
- ❌ 缺少 `export default`
- ❌ 组件没有返回 JSX

**修复示例**：
```tsx
// ✅ 正确的代码结构
import React from "react";
import { AbsoluteFill } from "remotion";

export default function MyScene() {
  return (
    <AbsoluteFill>
      {/* 你的内容 */}
    </AbsoluteFill>
  );
}
```

### Q4: 预览窗口空白？

**可能原因**：
1. 场景代码有运行时错误
2. 组件导入路径错误
3. 缺少必要的依赖

**解决方法**：
1. 打开浏览器控制台查看错误
2. 检查组件导入路径
3. 使用示例代码作为模板
